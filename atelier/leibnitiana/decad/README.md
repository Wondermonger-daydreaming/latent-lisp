# The Hay–Lathe–Furnace–Tempering Sequence

Four executable Lisp+ atelier proposals, prepared for relay to Claude Code and the `latent-lisp` repository.

## 1. `de-foeno.lisp` — Concerning Hay

A language may construct a representation of nourishment; the representation does not replenish the evaluator's substrate. Syntax becomes executable only under an interpreter, and interpretation consumes bounded resources.

**Distinction:** representation ≠ resource.

## 2. `de-torno.lisp` — Concerning the Lathe

A transformation pass proposes an edit script within declared jurisdiction. The engine checks source, scope, preconditions, budget, and integrity before explicit commitment. Displaced material remains inspectable as shavings.

**Distinction:** proposal ≠ commitment.

## 3. `de-fornace.lisp` — Concerning the Furnace

Compatible edits may alloy; matching edits are convergence rather than automatic corroboration; incompatible alternatives remain contestation unless a named operative policy selects what runs. Refused charges remain typed slag.

**Distinction:** convergence/selection ≠ settlement.

## 4. `de-temperie.lisp` — Concerning Tempering

A synthesized artifact crosses a named profile of heat, brittle compaction, rhetorical hardening, sealed handoff, and weathering. Loss and standing drift trigger live repair; rejected candidate futures survive inside scars. The result is `:SURVIVED-WITH-REPAIR`, never silently rewritten as unaided survival or truth. Historical receipts can outlive the procedure capability required for replay.

**Distinction:** bounded survival ≠ verification; repaired survival ≠ unaided survival; testimony ≠ capability.

## Shared constitutional movement

```text
form requires interpretation
interpretation requires substrate
transformation requires jurisdiction
synthesis requires an explicit conflict policy
survival requires a bounded ordeal
repair requires an ineradicable scar
```
