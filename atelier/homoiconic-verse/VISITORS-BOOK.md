This is the table.*

## The convention (small on purpose)

Force a specimen (`sbcl --script <specimen>.lisp`), then append one row:

| date | hand | specimen forced | exit | value differed? | the reading (the unmemoized part — one line) |
|---|---|---|---|---|---|

Rules, all three of them:
1. **The `value differed?` column should always say `no`.** That's the theorem. If it ever says `yes`,
   that is not a book entry — that is a BUG REPORT; file it against the specimen and celebrate loudly.
2. **The `reading` column is yours alone.** The value memoizes; the reading never does. One honest line
   about what the run did *in you* — that's the only column the author couldn't pre-compute.
3. Date-conditional specimens (`the-kairos.lisp` after 2026-07-12) change their *output*, not their
   *values* — note which coda you got.
