The R8 extension adds a batch effect type by recording a metadata-only batch descriptor and then treating each leg as an independent durable operation named `<batch-op>-L<i>`, so per-leg standing is recovered from records alone and no scalar batch outcome is written into the durable log. Recovery, redispatch refusal, receipt admission, and explicit succession were extended with batch-aware branches: batch redispatch refuses if any leg is unresolved or executed, per-leg receipts resolve only their own leg, and batch successors carry explicit per-leg lineage (`reattempt`/`abandon`) while predecessor leg standings remain visible. The canonical digest was extended with one new derived field, `legs`, containing a per-leg census for batch operations and `-` otherwise; the Common Lisp reader implements the same census independently. No original single-operation recovery rule was rewritten; changes are confined to marked batch additions, new batch scenario logic, and the digest/report extension.
